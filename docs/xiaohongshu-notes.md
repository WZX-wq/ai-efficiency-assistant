### 笔记 5：React + TypeScript 开发 Chrome 扩展，保姆级教程

**封面文案**：React开发Chrome扩展教程

**正文**：
谁说 Chrome 扩展只能用原生 JS 写？用 React + TypeScript 一样可以，而且开发体验好 10 倍！💪

我最近用 React + TypeScript + Vite 开发了一个 AI 效率助手 Chrome 扩展，从零到上架，踩了不少坑，今天把经验分享给大家。

**技术栈选择**：
⚛️ React 18 — 组件化开发，UI 构建效率高
📘 TypeScript — 类型安全，减少运行时 Bug
⚡ Vite 5 — 构建速度快，配合 @crxjs/vite-plugin 超方便
🎨 Tailwind CSS — 快速写样式

**Chrome 扩展的三个核心部分**：

1️⃣ **Background Script（Service Worker）**
后台运行的脚本，处理右键菜单、API 调用、消息路由

2️⃣ **Content Script**
注入到网页中的脚本，负责选中文字检测、浮动按钮渲染

3️⃣ **Popup**
点击扩展图标弹出的窗口，用于设置和手动操作

**开发流程**：
```
初始化项目 → 配置 Manifest V3 → 开发各模块 → 本地调试 → 打包 → 上架
```

**踩坑提醒**：
⚠️ Manifest V3 不支持远程代码，所有代码必须打包在扩展里
⚠️ Content Script 的 CSS 会和页面样式冲突，注意隔离
⚠️ Chrome Storage API 是异步的，别忘了用 Promise 封装

完整源码我已经开源了，GitHub 搜「ai-efficiency-assistant」，包含详细的中文注释，新手也能看懂。

**使用步骤**：

**Step 1：注册账号**
访问白山智算官网，手机号注册，2 分钟搞定

**Step 2：获取 API Key**
进入控制台 → API Keys → 创建新 Key
⚠️ Key 只显示一次，记得复制保存！

**Step 3：调用 API**
白山智算兼容 OpenAI 格式，代码超简单：

```javascript
const response = await fetch('你的API地址', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': 'Bearer 你的API Key'
  },
  body: JSON.stringify({
    model: 'deepseek-chat',
    messages: [{ role: 'user', content: '你好' }]
  })
});
```

**Step 4：处理响应**
返回的数据格式和 OpenAI 完全一样，直接解析就行

**费用说明**：
💰 DeepSeek-V3：百万 token 只需要几块钱
